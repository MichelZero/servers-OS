Aprenda como contribuir para o nosso Projeto Exemplo no GitHub!

Bem-vindo(a) ao Guia de Contribuição!

Este guia irá te orientar passo a passo sobre como contribuir para o nosso Projeto Exemplo hospedado no GitHub. Contribuir para projetos open source é uma ótima forma de aprender, ganhar experiência e ajudar a comunidade.

Link do Repositório Exemplo: https://github.com/SEU-USUARIO-OU-ORG/projeto-exemplo

Importante: Substitua SEU-USUARIO-OU-ORG/projeto-exemplo pelo link real do seu projeto no GitHub!

Pré-requisitos

  • Uma conta no GitHub.
  • Git instalado na sua máquina. Se não tiver, baixe aqui.
  • Um editor de código de sua preferência (VS Code, Sublime Text, Atom, etc.).
  • Conhecimento básico de Git e GitHub (fork, clone, branch, commit, push, pull request).

Passos para Contribuir

1. Faça um Fork do Repositório

O primeiro passo é criar uma cópia ("fork") do repositório original para a sua conta no GitHub. Isso permite que você trabalhe em sua própria versão do projeto sem afetar o original.

  1. Acesse a página do Projeto Exemplo no GitHub.
  2. Clique no botão "Fork" no canto superior direito da página.
  3. Selecione sua conta do GitHub para onde o repositório será forkado.

2. Clone o Repositório Forkado

Agora, clone o repositório que você acabou de "forkar" para a sua máquina local. Abra seu terminal ou Git Bash e use o comando abaixo, substituindo SEU-USUARIO-GITHUB pelo seu nome de usuário no GitHub. 

git clone https://github.com/SEU-USUARIO-GITHUB/projeto-exemplo.git

Navegue para o diretório do projeto clonado:

cd projeto-exemplo

3. Crie uma Nova Branch

É uma boa prática criar uma nova "branch" (ramificação) para cada nova funcionalidade ou correção de bug. Isso mantém suas alterações isoladas e organizadas. Escolha um nome descritivo para sua branch. 

git checkout -b nome-da-sua-branch

Exemplos de nomes de branch:

  • feature/adicionar-nova-funcionalidade
  • fix/corrigir-bug-no-login
  • docs/atualizar-documentacao

4. Faça suas Alterações

Agora é hora de trabalhar! Abra o projeto no seu editor de código e faça as alterações necessárias. Pode ser adicionar um novo recurso, corrigir um bug, melhorar a documentação, etc.

5. Faça Commit das Suas Alterações

Após fazer suas alterações, adicione os arquivos modificados ao "stage" e faça um "commit" com uma mensagem clara e descritiva. 

git add .  # Adiciona todos os arquivos modificados
# Ou git add nome-do-arquivo-especifico.js

git commit -m "Descreva suas alterações aqui de forma concisa"

Boas práticas para mensagens de commit:

  • Seja breve e objetivo no título (primeira linha).
  • Use o imperativo (ex: "Corrige bug" em vez de "Corrigido bug").
  • Se necessário, adicione uma descrição mais detalhada após uma linha em branco.

6. Envie (Push) suas Alterações para o Seu Fork

Envie as alterações da sua branch local para o seu repositório forkado no GitHub. 

git push origin nome-da-sua-branch

7. Crie um Pull Request (PR)

Agora que suas alterações estão no seu fork no GitHub, você pode propor que elas sejam incorporadas ao repositório original. Isso é feito através de um "Pull Request" (PR).

  1. Vá para a página do seu repositório forkado no GitHub (https://github.com/SEU-USUARIO-GITHUB/projeto-exemplo).
  2. Você verá uma notificação sugerindo a criação de um Pull Request para a branch que você acabou de enviar. Clique em "Compare & pull request".
  3. Se não vir a notificação, vá para a aba "Pull requests" e clique em "New pull request".
  4. Certifique-se de que a "base repository" é o repositório original (SEU-USUARIO-OU-ORG/projeto-exemplo) e a "base branch" é geralmente main ou master.
  5. A "head repository" deve ser o seu fork e a "compare branch" deve ser a sua nome-da-sua-branch.
  6. Adicione um título claro e uma descrição detalhada para o seu PR, explicando o que você fez e por quê. Se houver issues relacionadas, mencione-as (ex: "Closes #123").
  7. Clique em "Create pull request".

8. Aguarde a Revisão e Merge

Os mantenedores do projeto irão revisar seu Pull Request. Eles podem fazer comentários, solicitar alterações ou aprovar e fazer o "merge" (incorporar) das suas contribuições. Fique atento às notificações do GitHub e responda a quaisquer feedbacks.

Parabéns! Você fez sua primeira contribuição!

Dicas Finais

  • Comunicação: Se tiver dúvidas, abra uma "Issue" no repositório original antes de começar a trabalhar em algo grande.
  • Mantenha seu fork atualizado: Antes de criar uma nova branch, sincronize seu fork com o repositório original para evitar conflitos. 
    # Adicionar o repositório original como "upstream" (só precisa fazer uma vez)
git remote add upstream https://github.com/SEU-USUARIO-OU-ORG/projeto-exemplo.git

# Buscar as atualizações do upstream
git fetch upstream

# Ir para sua branch principal (geralmente main ou master)
git checkout main

# Mesclar as atualizações do upstream/main na sua main local
git merge upstream/main

# Enviar as atualizações para sua main no GitHub
git push origin main
  • Siga as diretrizes de contribuição: Muitos projetos têm um arquivo CONTRIBUTING.md com regras e diretrizes específicas. Leia-o!
  • Escreva testes: Se o projeto utiliza testes, adicione testes para suas novas funcionalidades ou correções.
