Configurando o Corretor Ortográfico em Português no VS Code

Este tutorial mostra como configurar a verificação ortográfica para o idioma português no Visual Studio Code usando a extensão Code Spell Checker. O processo é simples e melhora muito a qualidade de textos, comentários e documentação em seus projetos.

Passo 1: Instale a Extensão Principal "Code Spell Checker"

Esta é a extensão base que realiza a verificação ortográfica.

  • Abra o VS Code e acesse a aba de Extensões na barra lateral (Ctrl+Shift+X).
  • Na barra de pesquisa, digite Code Spell Checker.
  • Clique em Instalar na extensão publicada por Street Side Software.

Imagem da extensão "Code Spell Checker" na loja do VS Code.

Passo 2: Instale o Dicionário de Português

Para adicionar o suporte ao português, você precisa instalar um dicionário específico.

  • Ainda na aba de Extensões, pesquise por Brazilian Portuguese - Code Spell Checker.
  • Instale a extensão correspondente, também da Street Side Software. Isso adiciona o dicionário pt-BR ao corretor.

Imagem da extensão de dicionário português na loja do VS Code.

Passo 3: Habilite o Dicionário de Português

Após a instalação, certifique-se de que o dicionário está ativo para seu ambiente de trabalho.

  • Pressione Ctrl+Shift+P para abrir a Paleta de Comandos.
  • Digite Spell Checker Configuration Info e selecione a opção.
  • Na nova aba, marque a caixa correspondente a portuguese-brazil ou pt_BR para habilitá-lo.

Opção Avançada: Configuração via JSON

Para um controle mais refinado, você pode editar diretamente os arquivos de configuração JSON. Esta é a abordagem recomendada para projetos específicos.

1. Configuração Global (settings.json)

Esta configuração se aplica a todos os seus projetos. Abra o arquivo settings.json (Ctrl+Shift+P > Preferences: Open User Settings (JSON)) e adicione a seguinte linha:

// Dentro do seu settings.json
{
  "cSpell.language": "en,pt,pt_BR"
}

Isso permite a verificação simultânea de inglês e português.

2. Configuração por Projeto (cspell.json)

Esta é a melhor prática para projetos, pois não afeta suas configurações globais. Crie um arquivo chamado cspell.json na raiz do seu projeto:

{
  "version": "0.2",
  "language": "en,pt,pt_BR",
  "words": [
    // Adicione aqui palavras personalizadas do projeto
    "meuFramework",
    "minhaVariavel"
  ],
  "ignorePaths": [
    "node_modules/**",
    "dist/"
  ]
}

O arquivo cspell.json tem prioridade sobre as configurações globais e é ideal para compartilhar as regras do corretor com a equipe do projeto.
