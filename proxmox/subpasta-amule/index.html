Guia: Compartilhando Arquivos de Subpastas no aMule

Mantenha seus arquivos organizados sem perder o compartilhamento na rede.

O Problema: aMule Não Encontra Arquivos em Subdiretórios

Você organizou seus arquivos completos em subpastas dentro do diretório /Incoming, mas ao verificar a aba "Shared Files" no aMule, apenas os arquivos que estão na raiz do diretório aparecem. As pastas e seu conteúdo são completamente ignorados.

Por que isso acontece?

Por padrão, o aMule (assim como o eMule original) é projetado para escanear apenas o primeiro nível dos seus diretórios compartilhados. Ele não busca arquivos recursivamente dentro de subpastas. Essa é uma característica de design do software.

A Solução: Adicionar Subpastas como Diretórios Compartilhados

A boa notícia é que a solução é simples e não exige que você mude sua organização. Podemos instruir o aMule a tratar cada subpasta de interesse como um diretório compartilhado independente, editando diretamente seu arquivo de configuração.

Passo a Passo para a Configuração

Para que o aMule reconheça os arquivos dentro de alf-1-temporada e alf-2-temporada, siga estas etapas:

  1. Pare o aMule Daemon: É crucial parar o serviço antes de editar o arquivo de configuração para evitar que suas alterações sejam sobrescritas. 
    pkill -u amule
  2. Edite o arquivo amule.conf: Abra o arquivo de configuração com um editor de texto como o nano. 
    nano /home/amule/.aMule/amule.conf
  3. Adicione os Novos Diretórios: Navegue até a seção [eDonkey]. Abaixo das suas entradas existentes IncomingDir e TempDir, adicione uma nova linha SharedDirectory= para cada subpasta que deseja compartilhar. 
    [eDonkey]
...
IncomingDir=/mnt/downloads/Incoming
TempDir=/mnt/downloads/Temp
...

# ADICIONE AS LINHAS ABAIXO
SharedDirectory=/mnt/downloads/Incoming/alf-1-temporada
SharedDirectory=/mnt/downloads/Incoming/alf-2-temporada

    Você pode adicionar quantas linhas SharedDirectory= forem necessárias.

  4. Salve e Saia: No nano, pressione Ctrl+X, depois Y para confirmar e Enter para salvar.
  5. Reinicie o aMule: Inicie o daemon e a interface web novamente. 
    sudo -u amule amuled -f
sudo -u amule amuleweb
  6. Verifique o Resultado: Acesse a interface web, vá para a aba "Shared Files" e clique no botão de recarregar (Reload). Os arquivos de dentro das subpastas agora deverão aparecer na lista, prontos para serem compartilhados!

Agora você pode manter sua biblioteca organizada e totalmente compartilhada.
